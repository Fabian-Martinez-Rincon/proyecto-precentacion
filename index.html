ORM significa "Object-Relational Mapping" en inglés, y se refiere a una técnica de programación utilizada en el desarrollo de software. ORM es un acrónimo que desglosa de la siguiente manera:

O

  • Object (Objeto): Hace referencia a objetos en la programación orientada a objetos (POO). Los objetos son instancias de clases que contienen datos (atributos) y comportamientos (métodos).

R

  • Relational (Relacional): Se refiere a bases de datos relacionales, que son sistemas de gestión de bases de datos que almacenan datos en tablas con filas y columnas. Las bases de datos relacionales utilizan un lenguaje llamado SQL (Structured Query Language) para gestionar datos.

M

  • Mapping (Mapeo): En el contexto de ORM, "mapping" significa la correspondencia o relación entre objetos en la programación orientada a objetos y tablas en una base de datos relacional.

Para que nos sirve?

Ventajas de ORM:

  • Abstracción de la base de datos.
  • Portabilidad de la base de datos.
  • Aumento de la productividad.
  • Mantenimiento simplificado.

Desventajas de ORM:

  • Posible impacto en el rendimiento.
  • Menos control sobre las consultas SQL.
  • Curva de aprendizaje.
